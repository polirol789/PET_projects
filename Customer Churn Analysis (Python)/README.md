# Аналіз відтоку клієнтів (Customer Churn Analysis) & RFM-сегментація

[🔗 Переглянути код проєкту (Jupyter Notebook)](посилання_на_твій_файл_тут)

## 🎯 Бізнес-проблема
**Утримання клієнтів (Retention)** коштує в 5-10 разів дешевше, ніж залучення нових. У цьому проєкті я досліджую поведінку користувачів сервісу, щоб виявити "червоні прапорці" — ознаки того, що клієнт збирається піти, та надати бізнесу конкретні рекомендації щодо їх утримання.

## 🛠 Технічний стек
* **Інструменти:** Python (Pandas для обробки, Seaborn/Matplotlib для візуалізації, Scikit-learn — логістична регресія).
* **Методологія:** RFM-аналіз (Recency, Frequency, Monetary), кореляційний аналіз, когортний аналіз.
* **Дані:** Датасет з інформацією про транзакції, тип контракту та активність клієнтів.

## ⚙️ Етапи роботи
1. **Data Audit та очищення:** Видалення порожніх CustomerID, фільтрація повернень та нульових цін, розрахунок загального доходу (Revenue).
2. **Customer-level агрегація:** Розрахунок метрик Recency, Frequency, Monetary, а також Lifetime та Average Order Value.
3. **Churn definition:** Визначення цільової змінної (відтік = відсутність покупок понад 180 днів).
4. **RFM-сегментація:** Розподіл клієнтів на групи (Champions, At Risk, Lost тощо) для персоналізації підходу.
5. **Retention / Cohort analysis:** Аналіз поведінки клієнтів у часі та виявлення етапу, на якому вони йдуть.
6. **Predictive modeling:** Побудова логістичної регресії для визначення драйверів Churn та розрахунку ймовірності відтоку.

## 🧩 Виклики та рішення
* **Missing Data (~25% транзакцій без ID):** Видалено неідентифіковані записи, щоб уникнути викривлення поведінкових метрик реальних клієнтів.
* **Повернення та аномалії:** Очищено дані від від’ємних значень, щоб отримати коректний показник Monetary.
* **Масштаб даних:** Використано `StandardScaler` для нормалізації ознак, що дозволило моделі коректно враховувати і частоту покупок, і суму витрат.
* **Дисбаланс класів:** Оцінка моделі проводилася не лише за точністю, а й за ймовірністю (`predict_proba`), що критично для бізнес-рішень.

## 📈 Головні інсайти
* **Retention:** Найкритичніший період — перші 1–2 місяці після першої покупки, де спостерігається найбільший спад активності.
* **Ключовий фактор:** Показник `recency_days` (свіжість покупки) є найсильнішим індикатором відтоку.
* **Регулярність vs Гроші:** Частота покупок (`frequency`) має значно більший "захисний ефект", ніж сума середнього чека.

## 💡 Рекомендації для бізнесу
1. **Активація новачків:** Впровадити вітальну серію оферів у перші 30–60 днів для стимулювання другої покупки.
2. **Тригерні комунікації:** Автоматизувати розсилки для клієнтів, чий показник Recency перевищує середній по їхньому сегменту.
3. **Лояльність:** Спрямувати зусилля на програми, що підвищують частоту візитів, оскільки це головний чинник утримання.
4. **Пріоритезація:** Використовувати передбачену ймовірність відтоку для фокусування зусиль відділу підтримки на сегменті "At Risk" з високим потенціалом повернення.
