## 📌 Опис проєкту
Цей проєкт присвячений аналізу даних про товари IKEA з використанням Python.  
Основна мета — провести очищення даних, дослідницький аналіз (EDA), виявити закономірності та перевірити статистичні гіпотези.

Дані взяті з відкритого датасету **TidyTuesday (03.11.2020)**.

---

## 📂 Джерело даних
- **Dataset:** IKEA products dataset  
- **Source:** https://github.com/rfordatascience/tidytuesday/tree/main/data/2020/2020-11-03  

---

## 🛠️ Використані технології
- Python  
- Pandas, NumPy  
- Matplotlib, Seaborn  
- SciPy  
- Scikit-learn  

---

## 🔄 Етапи роботи

### 1. Завантаження даних
- Автоматичне завантаження CSV через `requests`  
- Збереження локальної копії  

### 2. Первинний аналіз
- Розмірність датасету  
- Типи даних  
- Описова статистика  
- Перевірка пропущених значень  

---

## 🧹 Очищення даних
Було виконано:

- Перетворення `old_price` у числовий формат  
- Видалення неінформативних колонок:
  - `Unnamed: 0`  
  - `link`  
  - `short_description`  
- Очищення поля `designer`  
- Видалення:
  - дублікатів по `item_id`  
  - сутнісних дублікатів (за ключовими ознаками)

## 🧪 Перевірка гіпотез

### 🔹 Гіпотеза 1
**Товари з більшою кількістю кольорів дорожчі**

**Методи:**
- Тест Шапіро-Вілка (нормальність)  
- Кореляція Спірмена  
- t-test  

📌 **Висновок:**  
Є слабкий позитивний зв’язок між кількістю кольорів і ціною.

---

### 🔹 Гіпотеза 2
**Товари однієї категорії мають схожу ціну**

**Методи:**
- ANOVA  
- Коефіцієнт варіації  
- Кластеризація (KMeans)  

📌 **Висновок:**
- Різниця між категоріями статистично значуща  
- Деякі категорії мають високу варіативність цін  

---

## 🤖 Кластеризація
- Використано KMeans  
- Підбір кількості кластерів методом *elbow*  
- Групування категорій за ціною  

---

## 📌 Висновки
- Ціна товарів залежить від категорії та характеристик  
- Кількість кольорів має слабкий вплив  
- Дані мають значну варіативність у різних категоріях  
- Кластеризація дозволяє виділити групи товарів за ціновими сегментами
